服务-持久化要点

  • 组件之间无法相互调用，如果想要相互调用的话，需要使用服务，服务是公共化的【具体组件间的关系请看后面父子组件的交互】

  • 组件可以调用服务的方法，但服务不能调用组件的方法，并且组件之间不能相互调用

  • 服务之间可以相互调用

Observable发出数据，subscirbe接收数据

首先在服务组件中引入rxjs对象

import { Observable } from "rxjs";

在服务中写好调用异步数据的方法

getRxjsData(){

  return new Observable((observer) =>{

   setTimeout(() => {

​    var username='张三';

​    observer.next(username);//用它来发布成功数据

​    // observer.error()

​    // 用它来返回失败数据

   }, 1000);

  })

 }

在需要调用数据的组件中，第一步写服务的调用

import { RequestService } from "../../service/request.service";

constructor(public request:RequestService)

第二步在构造方法中继续获取数据

var rxjsData = this.request.getRxjsData();

  rxjsData.subscribe((data) => {

   console.log(data)

  })

这样就可以在控制台看到数据被打印

Rxjs还提供了更多的方法，比如可以取消订阅

//过一秒后，撤回刚才的操作

var stream = this.request.getRxjsData();

  var d = stream.subscribe((data) => {

   console.log(data);

  })

  setTimeout(() => {

   d.unsubscribe(); //取消订阅

  }, 1000);

或者多次执行订阅

在request.service.ts中

// 多次执行

getRxjsIntervalData(){

 let count = 0;

 return new Observable((observer) =>{

  setInterval(() => {

   count++;

   var username='张三--interval' + count;

   observer.next(username);

   // observer.error()

   // 用它来返回失败数据

  }, 1000);

 })

}

在组件中

//多次执行

  var streaminterval = this.request.getRxjsIntervalData();

  streaminterval.subscribe((data) => {

   console.log(data);

  })

Rxjs还有许多工具函数，以map和filter为例

在用到的组件中引入

import { map, filter } from "rxjs/operators";

在组件方法中

  var streamNum = this.request.getRxjsIntervalNum();

  streamNum.pipe(

   filter((value:any) =>{

​    if (value % 2 == 0) {

​     return true;

​    }else return false;

   })

  )

  .subscribe((data) => {

   console.log(data);

  })

这里的filter中一定要声明value是any类型，否则会报错

  })

也可以用在一起(通过pipe管道流水线处理)

3、通过jsonp请求（解决跨域的方案）

注意，如果我们要使用jsonp，需要确认服务器支持，如何查看呢？

在请求链接后端添加?callback=xxx,xxx的地方随便写，你会看到请求的数据前后有了xxx字样，代表它支持

post的请求有个缺点，就是它如果存在跨域，服务器可能因为策略拒绝连接，用jsonp可以解决这个问题

首先，在app.module.ts中引入HttpClientModule 包和httpClientJsonpModul包

import { HttpClientModule, HttpClientJsonpModule } from “@angular/common/http”;

同样，还需要声明

imports: [HttpClientModule, HttpClientJsonpModule]

在模组中同样和http的声明方式一样，在构造方法中声明http对象即可

下面就可以开始使用jsonp

doJsonpLogin(){

let api=”http://a.itying.com/api/productlist“;

this.http.jsonp(api,’callback’).subscribe((response) =>{

console.log(response)

})

}

angular的路由

l 配置路由功能和进行路由

在你新建项目的时候，记得选择添加路由就可以自动配置

然后在app-routing.module.ts中配置下面的内容

1、首先引入需要路由的组件，比如我们需要home组件：

import { HomeComponent } from ‘./components/home/home.component’;

3、 在routers列表中写入，其中path后是用户访问的地址

const routes: Routes = [

{path:’home’,component:HomeComponent}

];

这里注意path的位置不要写多于内容，我就是有一次写成了/home，多了斜杠，页面就什么也显示不出来，找了好久问题

配置好后，我们就可以在浏览器中按链接访问每一个地址：（以news组件为例）

接下来，我们看看如何使用a标签进行导航，有三种方式，推荐第一种：

<a [routerLink]="[ '/home' ]">首页标签</a>//推荐

<a routerLink='/news'>新闻标签</a>

<a href="/product">商品页面</a>

第一种是动态绑定，第二种是静态，第三种则是原生方法

但在此我们又发现了一个问题，进来后第一个页面除了导航栏没有任何内容，干巴巴的，怎么办呢？可以让angular默认跳转一个页面

{path:’**’,redirectTo:’home’}

这个就代表是没有找到路由地址的时候加载的组件，其中要注意的是redirectTo需要的参数是string类型的，它需要路径**而不是组件名

angular提供了routerLinkActive的方法，举个例子：

<a [routerLink]=”[ ‘/home’ ]” routerLinkActive=”active”>首页标签

在刚才的基础上加上routerLinkActive标签即可，其中的active位置是你的css标签类，他会在css中寻找名为active类的样式为其装扮，就像这样

l 动态路由及其跳转传值

想象这样一个场景，在商品列表中选中一个商品，进入详情页。这个步骤中我们需要知道你选中的是那个产品，并跳转到相应页面，而且我们还需要能够返回列表。angular中有这两种方式来完成

1、get传值

首先将标签改成这样

​ <a [routerLink]=”[ ‘/newscontent’ ]” [queryParams]=”{aid:key}”>—

这个queryParams是设置请求时的参数，参数是之前设置好的ngfor中的参数，我们看一下它请求的链接

http://localhost:4200/newscontent?aid=1

这样就知道了用户在请求哪一条数据

接着要让newscontent组件获得这个queryParams，需要在newscontent组件中引入

import { ActivatedRoute } from “@angular/router”;

constructor(public route:ActivatedRoute) { }

这样你就能在newscontent中获得这个参数

ngOnInit(): void {

this.route.queryParams.subscribe((data:any) =>{

console.log(data.aid)

}) }

2、动态路由

配置，后面的:aid就是动态部分（注意不要忘了前面的冒号）

#app-routing.module.ts

{path:’newscontent/:aid’,component:NewscontentComponent},

接下来，我们在网页标签中就可以直接使用

<a [routerLink]=”[ ‘/newscontent/‘, key ]”>—

这句话的格式是这样的，注意这里的path已经发生了改变，如果你还想要newscontent的跳转，要再多加一条，也就是说，保证一个路由一个链接

<a [routerLink]=”[ ‘/path’, routeParam ]”>name

routeParam就是我们的动态传值参数，url效果如下

http://localhost:4200/newscontent/6

有了这个我们就可以直接让用户去想去的地方

另外，动态路由中获取传值也和get方法相似，只是方法略有不同

//动态路由获取传值

this.route.params.subscribe((data:any) =>{

console.log(data.aid)})

<button (click)=”goNewsContent()”>get传值js跳转

import { Router, NavigationExtras } from “@angular/router”;

l 父子路由

顾名思义，就是在路由的界面中再添加一层路由，实现二级页面的跳转

模组引入肯定在定义父路由就做过了，所以我们直接下一步

在路由界面先引入子路由的页面

import { WelcomeComponent } from “./components/home/welcome/welcome.component”;

​ 设置路由，使用children属性（标红部分）

const routes: Routes = [{

path:’home’,component:HomeComponent,

children:[

{path:’welcome’,component:WelcomeComponent},

{path:’settings’,component:SettingsComponent},

{path:’**’,redirectTo:’welcome’} //设置默认路由，后面会说到

]}

设置好后，我们就可以在父页面中使用

<a [routerLink]=”[ ‘/product/pcate’ ]”>商品分类列表

当然，还需要在想让它出现的位置为它设置出口

这个时候，你的功能就实现了，不过你肯定发现了一个问题，正常的网页在访问时，（比如我当前访问的官网），我点了文档进入页面后立刻显示了第一个三级页面，看起来非常自然，而我们却不行，这里的解决方式和我们之前的一样，在子路由的数组中添加默认路由即可

就像这样：

const routes: Routes = [{

path:'home',component:HomeComponent,

​    children:[

​      {path:'welcome',component:WelcomeComponent},

​      {path:'**',redirectTo:'welcome'}//设置默认路由

] }
