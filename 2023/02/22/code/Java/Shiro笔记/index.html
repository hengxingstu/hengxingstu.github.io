Shiro

搭建基础环境

所需环境为

  • spring
  • springMVC
  • Shiro

步骤闪回

  1. 创建工程，引入spring-webmvc依赖

  2. 创建web工程的WEB-INF/web.xml

  3. 配置applicationContext.xml，这是spring的配置，使用默认过滤器，排除controller

  4. 配置spring-servlet.xml，这是springMVC的配置，不使用默认过滤器，在此加载controller。

    <mvc:annotation-driven/>配置映射器适配器

  5. 配置web.xml加载刚刚的配置，将其生效

    <!--加载spring配置文件-->
<context-param>
    <param-name>contextConfigLocation</param-name>
    <param-value>classpath:applicationContest.xml</param-value>
</context-param>
<listener>
    <listener-class>org.springframework.web.context.ContextLoaderListener</listener-class>
</listener>
<!--加载servlet-->
<servlet>
    <servlet-name>springMVC</servlet-name>
    <servlet-class>org.springframework.web.servlet.DispatcherServlet</servlet-class>
    <init-param>
        <param-name>contextConfigLocation</param-name>
        <param-value>classpath:spring-servlet.xml</param-value>
    </init-param>
</servlet>
<servlet-mapping>
    <servlet-name>springMVC</servlet-name>
    <url-pattern>/</url-pattern>
</servlet-mapping>

  6. 添加controller测试

  7. 添加Shiro依赖

    <!-- shiro依赖 -->
<!-- 截止2023年2月22日，最新版本 -->
<dependency>
    <groupId>org.apache.shiro</groupId>
    <artifactId>shiro-web</artifactId>
    <version>1.11.0</version>
</dependency>
<!-- shiro和spring连接 -->
<dependency>
    <groupId>org.apache.shiro</groupId>
    <artifactId>shiro-spring</artifactId>
    <version>1.11.0</version>
</dependency>

    至此，环境搭建完成

控制器

@Controller
public class LoginController {

    @RequestMapping("/login")
    public String loginPage(){
        return "login";
    }

    @PostMapping("/doLogin")
    public String doLogin(String username, String password, Model model){
        //获取当前用户
        Subject subject = SecurityUtils.getSubject();
        UsernamePasswordToken token = new UsernamePasswordToken(username, password);

        try {
            subject.login(token);//执行登录操作，自动调用认证类Realm
            return "redirect:/hello";
        } catch (AuthenticationException e) {
            e.printStackTrace();
            model.addAttribute("error",e.getMessage());
        }
        return "forward:/login";
    }
}

自定义认证类

/**
 * 这个 Realm 相当于是我们自定义认证类
 * 当用户调用了 subject.login 方法之后，最终的调用逻辑就会来到当前类中
 */
public class MyRealm extends AuthenticatingRealm {

    HashMap<String, User> users = new HashMap<>();

    public MyRealm() {//初始化时直接添加用户信息进去
        User user1 = new User();
        user1.setUsername("cloud");
        user1.setPassword("1313");
        users.put("cloud", user1);
    }

    /**
     * 当用户执行登录操作的时候，最终就会调用到这个方法
     *
     * @param authenticationToken 这个参数中就包含了用户登录时输入的用户名密码
     * @return AuthenticationInfo 认证后的信息
     */
    @Override
    protected AuthenticationInfo doGetAuthenticationInfo(AuthenticationToken authenticationToken) throws AuthenticationException {
        UsernamePasswordToken usernamePasswordToken = (UsernamePasswordToken) authenticationToken;
        //获取用户登录时候输入的用户名
        String username = usernamePasswordToken.getUsername();
        //根据用户名去数据查询用户，假设这个就是取数据查询
        User user = users.get(username);
        //说明用户名写错了
        if (user == null) {
            throw new AuthenticationException("用户名错误");
        }
        //1. 用户登录的用户名
        //2. 用户的密码，但是注意，这个密码是从数据库查询出来的用户密码
        //这里返回之后，系统会自动根据这里返回的信息，再结合用户登录时输入的Token（用户名密码信息）去判断密码是否正确
        return new SimpleAuthenticationInfo(user.getUsername(), user.getPassword(), getName());
    }
}

        <filter-class>org.springframework.web.filter.DelegatingFilterProxy</filter-class>
    </filter>
    <filter-mapping>
        <filter-name>shiroFilter</filter-name>
        <url-pattern>/*</url-pattern>
    </filter-mapping>

错误页面

另外，未成功登录时，访问任何页面，都会强制跳转到登录页面要求登录。

不过现在这个项目还有两个小问题：

  1. 项目启动时，进入登录页面会自动在url中携带sessionid

    解决办法是：

    在配置SecurityManager时配置sessionManager属性，关闭 url 地址中的 sessionid 重写功能

    <!--2. 配置 SecurityManager-->
<bean class="org.apache.shiro.web.mgt.DefaultWebSecurityManager" id="securityManager">
    <property name="realm" ref="myRealm"/>
    <property name="sessionManager">
        <bean class="org.apache.shiro.web.session.mgt.DefaultWebSessionManager">
            <!--不开启 url 地址中的 sessionid 重写功能，也就是 sessionid 只能放在 Cookie 中-->
            <property name="sessionIdUrlRewritingEnabled" value="false"/>
        </bean>
    </property>
</bean>

  2. 跳转登录页面时，总是跳转http://localhost:8080/login.jsp

    在过滤器中添加loginUrl属性

    <bean class="org.apache.shiro.spring.web.ShiroFilterFactoryBean" id="shiroFilter">
    <!--当用户访问了一个没有权限的接口的时候，展示这个地址给用户-->
    <property name="loginUrl" value="/login"/>

之前的认证已经不满足我们的需求了，重新定义认证类

@Component
public class DbRealm extends AuthenticatingRealm {
    @Autowired
    UserMapper userMapper;

    @Override
    protected AuthenticationInfo doGetAuthenticationInfo(AuthenticationToken token) throws AuthenticationException {
        UsernamePasswordToken usernamePasswordToken = (UsernamePasswordToken) token;
        //获取用户登录时候输入的用户名
        String username = usernamePasswordToken.getUsername();
        //根据用户名去数据查询用户，假设这个就是取数据查询
        User user = userMapper.getUserByUsername(username);
        //说明用户名写错了
        if (user == null) {
            throw new AuthenticationException("用户名错误");
        }
        //如果账户被禁用了
        if (!user.getEnabled()) {
            throw new LockedAccountException("账户被禁用了");
        }
        //1. 用户登录的用户名
        //2. 用户的密码，但是注意，这个密码是从数据库查询出来的用户密码
        //这里返回之后，系统会自动根据这里返回的信息，再结合用户登录时输入的用户名密码信息去判断密码是否正确
        return new SimpleAuthenticationInfo(user.getUsername(), user.getPassword(), getName());
    }
}

注销登录

在过滤规则中添加/logout=logout，后面的 logout 是注销登录过滤器的简称。添加之后，shiro会将这条请求交给注销登录过滤器，注销登录过滤器直接注销登录。

当然，你也可以自己写个controller来处理，不过记得调整过滤器中的过滤权限为认证/logout=authc

@RequestMapping("/logout")
public String logout(){
    Subject subject = SecurityUtils.getSubject();
    subject.logout();
    return "login";
}

密码加密

先通过加密算法得到hash值

public static void main(String[] args) {
    SimpleHash md5 = new SimpleHash("MD5", "123");
    System.out.println("md5.toString() = " + md5.toString());
    //md5.toString() = 202cb962ac59075b964b07152d234b70

将其存储在数据库中。

现在，如果不对项目进行更改，shiro会自动将我们的密码明文不做任何处理，直接与数据库中的密码进行比对。想要成功登录，必须使用加密后的密文来登录才行。

如何让shiro自动对我们输入的密码进行加密，再比对呢？

我们来一探究竟，在程序subject.login(token);执行登录操作的地方打上断点，定位到对比密码的片段，在AuthenticatingRealm中

/**
 * Asserts that the submitted {@code AuthenticationToken}'s credentials match the stored account
 * {@code AuthenticationInfo}'s credentials, and if not, throws an {@link AuthenticationException}.
 *
 * @param token the submitted authentication token
 * @param info  the AuthenticationInfo corresponding to the given {@code token}
 * @throws AuthenticationException if the token's credentials do not match the stored account credentials.
 */
protected void assertCredentialsMatch(AuthenticationToken token, AuthenticationInfo info) throws AuthenticationException {
    CredentialsMatcher cm = getCredentialsMatcher();//获取密码比对器
    if (cm != null) {
        if (!cm.doCredentialsMatch(token, info)) {
            //not successful - throw an exception to indicate this:
            String msg = "Submitted credentials for token [" + token + "] did not match the expected credentials.";
            throw new IncorrectCredentialsException(msg);
        }
        
//我们来看getCredentialsMatcher，只是返回了当前的比对器
public CredentialsMatcher getCredentialsMatcher() {
        return credentialsMatcher;
    }

进入到assertCredentialsMatch中后，获取密码对比器，默认获得的是SimpleCredentialMatcher，这个对比器的对比就是明文对比，所以需要更改获取到的对比器

也就是说，如果我们重写这个类中的getCredentialsMatcher()方法，将比对器返回为我们需要的，就可以正确比对了

@Override
public CredentialsMatcher getCredentialsMatcher() {
    return new HashedCredentialsMatcher("MD5");
}

再次尝试后，直接输入明文即可，系统会在后台加密对比。

多Realm配置

如果希望多种验证策略并存，可以使用该策略。

这个是配置多Realm的认证器，在applicationContext.xml中添加

<bean class="org.apache.shiro.authc.pam.ModularRealmAuthenticator" id="authenticator">
    <property name="realms">
        <list>
            <ref bean="myRealm"/>
            <ref bean="dbRealm"/>
        </list>
    </property>
    <property name="authenticationStrategy">
        <bean class="org.apache.shiro.authc.pam.AtLeastOneSuccessfulStrategy"/>
    </property>
</bean>

其中authenticationStrategy确认认证策略

  • AtLeastoneSuccessfulstrategy：至少有一个Realm认证成功，就算成功

  • ALLSuccessfulstrategy：所有Realm 都要认证成功才算成功

  • FirstSuccessfulStrategy：至少有一个Realm 认证成功，就算成功

    这个Bean中，有一个属性名为stopAfterFirstSuccess，表示是否在第一个bean 认证成功后，就不再执行后面的Realm了

    这个属性默认为false，即第一个ReamL认证成功后，后面的Realm还是会认证的，此时Firstsuccessfulstrategy和AtLeastonesuccessfulstrategy其实没有差别。

    如果将stopAfterFirstSuccess属性设置为true，那么对于Firstsuccessfulstrategy而言，第一个Realm 认证成功后，后面的Realm就不再执行了。

另外，将单realm的设置变为刚刚配置好的认证器

RememberMe功能

remeber-me即记住我功能，是我们在登录web系统时的常见勾选项。当我们登录一个web系统时除了输入常规的用户名、密码后还可以勾选记住我选项（假设该系统提供了该选项），此时假设用户名、密码输入正确那么系统将会在客户浏览器cookie中记录用户登录相关认证相关信息。实现的效果就是当我们下次再次访问该网站某些页面时无需再次登录。

————————————————
版权声明：本文为CSDN博主「罗汉爷」的原创文章，遵循CC 4.0 BY-SA版权协议，转载请附上原文出处链接及本声明。
原文链接：https://blog.csdn.net/IndexMan/article/details/118726437

我们在很多网站都见过这样的功能，一般在关闭浏览器后重写打开时，可以无需登录访问网站。不过，在遇到涉及数据修改时仍然要求再次登录认证，比如淘宝网在关闭浏览器后重新登陆，可以继续浏览商品，但如果需要购买，就必须再次登录。

Shiro中为我们提供这个功能。

为了展示，我们定义一个index页面，这个页面的权限设置为rememberMe（即登录后重启浏览器还可以访问）。

@GetMapping("/index")

    return "This is index page.You can access this page through Remember Me!";
}

<form action="/doLogin" method="post">
    <input type="text" name="username" value="Aerith">
    <input type="password" name="password" value="123">
    <input type="checkbox" value="on" name="rememberMe">记住我

@PostMapping("/doLogin")
public String doLogin(String username, String password, Model model,String rememberMe){
    //获取当前用户
    Subject subject = SecurityUtils.getSubject();
    UsernamePasswordToken token = new UsernamePasswordToken(username, password);

    try {
        if ("on".equals(rememberMe)){//在这里判断，如果rememberMe为on，就开启功能
            token.setRememberMe(true);
        }
        subject.login(token);
        return "redirect:/hello";
    } catch (AuthenticationException e) {
        e.printStackTrace();
        model.addAttribute("error",e.getMessage());
    }
    return "forward:/login";
}

完成，登录成功后，关闭浏览器再次访问http://localhost:8080/index无需登录，但要访问http://localhost:8080/hello则需重新登录。

RememberMe登录流程

  1. 登录的时候，传入用户名密码，登录成功之后，先对当前登录的用户进行序列化，序列化之后，得到一个 byte 数组，然后对这个 byte 数组进行加密（用的是 AES 对称加密，将来可以解密的），加密之后得到一个 byte 数组，但是这个 byte 数组无法直接展示出来，需要再进行一次 Base64 转码，就可以转为可读的字符串了，然后将这个字符串写入到 Cookie 中，并返回给浏览器。
  2. 以后每次请求的时候，系统都会自动携带上这个 Cookie，系统收到这个 Cookie 之后，首先从请求头中提取出 RememberMe 字符串，提取出来之后首先进行 Base64 解码，解码之后，再调用 AES 服务进行解密，揭秘之后得到一个 byte 数组，再将这个 byte 数组反序列化，得到登录的用户对象。

角色与权限

  • 权限：具体的一个数据操作，例如针对 user 表，可以有 user:select、user:delete、user:update、user:add 等权限。权限重点是强调对数据的操作，权限本身不涉及到任何业务相关的技术点。
  • 角色：角色往往是根据你的项目的业务来定义的一组权限的集合，例如有一个角色是产品研发部经理，这个角色本质上是一个权限的集合。
  • 正常来说，当用户访问某一个接口或者某一个方法的时候，首先根据当前用户的 id，查询到当前用户的角色，再根据当前用户的角色，查询到当前用户的权限，再去判断当前用户是否具备所需要的权限。
  • Shiro 中，既可以使用角色去做判断，也可以使用权限去做判断。

/**
 * 当需要确定用户具备哪些权限时，系统会自动调用当前方法来获取当前登录用户所具有的权限
 * @param principals 当前用户
 * @return 权限信息
 */
@Override
protected AuthorizationInfo doGetAuthorizationInfo(PrincipalCollection principals) {
    String username = (String) principals.getPrimaryPrincipal();
    SimpleAuthorizationInfo info = new SimpleAuthorizationInfo();
    List<String> permissions = userMapper.getPermissionsByUsername(username);
    info.addStringPermissions(permissions);
    return info;
}
