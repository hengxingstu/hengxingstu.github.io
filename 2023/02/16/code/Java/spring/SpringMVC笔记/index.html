工作流程

注解详解

@RequestMapping详解

注解加在方法上，访问路径就是http://localhost:8080/mvc02/hello

现在，一个方法就是一个接口，通过@RequestMapping注解来指定访问方法的路径

这个注解也可以放在类上面，标注在类上时，value 相当于一个命名空间，即访问该 Controller 下的任意方法都需要带上这个命名空间。

默认情况下，这个注解支持多个请求方法，我们可以通过 method 属性去指定该接口支持哪些方法，但是注意，如果这个注解放在类上面，不能指定 method 属性

@RequestMapping(value = "/hello", method = RequestMethod.GET)

指定当前方法通过GET请求处理，这时候就只能通过GET发送请求了。如果是其他方式会报错：Request method 'POST' not supported

RESTful风格

一个链接，根据请求方式的不同，有不同的功能

http://localhost:8080/mvc02/user/1

  • GET：查询id为1的user

  • POST：添加id为1的user

  • PUT：修改id为1的user

  • DELETE：删除id为1的user

我们可以指定method参数，但是这样还稍稍有点麻烦，spring为我们提供了简便的注解

@GetMapping("/hello")

其实他的实现方式就是内部封装了之前的写法而已

@Target({ElementType.METHOD})
@Retention(RetentionPolicy.RUNTIME)
@Documented
@RequestMapping(
    method = {RequestMethod.GET}
)
public @interface GetMapping

当然，四种请求方式都有其注解

  • @PostMapping()
  • @PutMapping()
  • @DeleteMapping()

接口方法的返回值

  1. 返回 ModelAndView

  2. 返回字符串

    • 字符串代表服务端或者客户端跳转

    • redirect:重定向

      @GetMapping("/hello2")
public String hello2(){
    return "redirect:http://www.baidu.com";
}

      注意，如果通过response.sendRedirect方法去写重定向，那么跳转路径中，需要包含项目名称

      如果通过 SpringMVC 中的字符串进行重定向，那么不需要写项目路径。例如转到http://localhost:8080/mvc02/home/hello，直接写return "redirect:/home/hello";

    • forward:

      @GetMapping("/hello3")
public String hello3(){
    return "forward:/home/hello2";
}

    • 字符串代表逻辑视图名

      @GetMapping("/hello")
public String Modelhello(Model model){
    model.addAttribute("username","hengxing");
    //这个表示是一个逻辑视图名，系统会去自动查找一个名为 index 的视图，并返回
    return "index";
}

      servlet会自动去找index.jsp的文件，这里的前缀后缀是你之前在视图解析器中写的。

    • 真的字符串

      @GetMapping("/hello4")
//这个注解表示方法的返回值不用在进行页面渲染了，当前方法的返回值，直接写回到浏览器
@ResponseBody
public String hello4(){
    return "index";
}

      直接在页面显示index字符

      刨根问底

      关于@ResponseBody 你可以查看资料：@ResponseBody注解的使用说明

  3. 返回 void

    @GetMapping("/hello5")
//方法没有返回值，也需要该注解，加了这个注解，方法就不会去找对应的视图了
@ResponseBody
public void hello5(){
}

接口的参数

  1. 默认可以直接注入的参数：

    • HttpServletRequest
    • HttpServletResponse
    • HttpSession
    • Model/ModelMap

@ResponseBody
public void hello6(HttpServletRequest request, HttpServletResponse response) throws IOException {
    String username = request.getParameter("username");
    System.out.println("username = " + username);
    response.getWriter().println(username);
}

    开箱即用，发送请求http://localhost:8080/mvcTest/hello6?username=hengxing

  2. 自定义的参数

请求参数的传递：

  1. 地址栏参数：http://localhost:8080/mvc02/user?name=xxx

    http://localhost:8080/mvcTest/book/getBook?name=简爱&author=夏洛蒂勃朗特

    获取时通过：request.getParameter()

  2. 参数放在请求体中

  • 可以是 key-value 的形式

    也可以在java中直接用写好的实体类进行接收，==参数名是根据getter、setter方法推断==的。

    @PostMapping(value = "/getBook2",produces = "text/html;charset=utf-8")
@ResponseBody
public void getBook2(Book book){
    System.out.println("book = " + book);
}

    获取时通过：request.getParameter()

  • 可以是 JSON 形式

    获取时通过：request.getInputStream()

  1. 参数可以放在地址栏，路径变量：http://localhost:8080/mvc02/user/1

    默认情况下{}中的变量名要和参数中的变量名一致。如果不一致，则可以通过@PathVariable中的属性进行配置

    @PostMapping("/findBook/{id}/{name}")
@ResponseBody
public void findBook(@PathVariable("id") Integer id, @PathVariable("name") String name){
    System.out.println("id = " + id);
    System.out.println("name = " + name);
}

    获取时通过：request.getPathI()

    <filter-class>org.springframework.web.filter.CharacterEncodingFilter</filter-class>
    <init-param>
        <param-name>encoding</param-name>
        <param-value>UTF-8</param-value>
    </init-param>
    <init-param>
        <param-name>forceRequestEncoding</param-name>
        <param-value>true</param-value>
    </init-param>
    <init-param>
        <param-name>forceResponseEncoding</param-name>
        <param-value>true</param-value>
    </init-param>
</filter>
<filter-mapping>
    <filter-name>encodingFilter</filter-name>
    <url-pattern>/*</url-pattern>
</filter-mapping>

在@PostMapping中添加produces参数，这里的作用只是页面显示时需要告诉浏览器如何解析，并不包含参数本身的字符编码

@PostMapping(value = "/getBook",produces = "text/html;charset=utf-8")
@ResponseBody
public String getBook(String name,String author){
    System.out.println("name = " + name);
    System.out.println("author = " + author);
    return name;
}

参数别名

刚刚的情况中，发送请求时的参数名，必须和方法中的形参名称一致，否则就找不到参数。

但是我们可以通过@RequestParam为这个参数设置别名，你可以看到，这里使用bookname做书名的别名

不过用了这个注解之后，默认情况下，==对应的参数必填，否则就会报错==

@PostMapping(value = "/getBook",produces = "text/html;charset=utf-8")
@ResponseBody
public String getBook(@RequestParam("bookname") String name, String author){
    System.out.println("name = " + name);
    System.out.println("author = " + author);
    return name;
}

可以通过required = false解决，它表示这个参数不是必填参数。或者设置一个默认值

我们添加了一个日期字段，希望通过前端表单页面接收日期参数，但是spring在日期转换时会报错。这时候就需要我们定义一个转换器进行转换。

如何定义转换器并应用到springMVC呢？

  1. 编写转换器类，实现org.springframework.core.convert.converter.Converter接口
  2. 注册进spring容器
    1. DateConverter注册
    2. conversionServiceFactoryBean注册，并把DateConverter放入其中
    3. mvc:annotation-driven中，开启conversion-service并将FactoryBean放入其中
@Component
public class DateConverter implements Converter<String, Date> {
    public Date convert(String source) {
        SimpleDateFormat format = new SimpleDateFormat("yyyy-MM-dd");
        try {
            return format.parse(source);
        } catch (ParseException e) {
            e.printStackTrace();
        }
        return null;
    }
}
<bean id="conversionServiceFactoryBean" class="org.springframework.format.support.FormattingConversionServiceFactoryBean">
    <property name="converters">
        <set>
            <ref bean="dateConverter"/>
        </set>
    </property>
</bean>
<!--
将 serviceFactoryBean 配置给 annotation-driven，
所有配置给 annotation-driven 的属性，最终其实都
是配置给 HandlerMapping 或者 HandlerAdapter
-->
<mvc:annotation-driven conversion-service="conversionServiceFactoryBean"/>

JSON参数处理

处理工具

  1. jackson，官方，Spring Boot 中默认的。
  2. gson，Google，安卓上使用较多。
  3. fastjson，Alibaba。

HttpMessageConverter

将对象转为 JSON 以及将 JSON 转为对象

JSON 处理的核心工具。

SpringMVC 中使用 JSON 实际上是两个方面：

  • 返回值，返回 JSON
  • 请求的参数是一个 JSON

如果是返回值，接口定义的时候，直接返回的是对象，所以需要有一个工具，把我们返回的对象转为 JSON，这个工具就是 HttpMessageConverter。

如果是请求参数，那么 JSON 一般是放在请求体中，我们需要一个工具，将请求体中的 JSON 字符串转为一个对象。

HttpMessageConverter 本身是一个接口，是一个规范。具体怎么做，要看不同的 JSON 处理工具。

如果使用的是 jackson 或者 gson，那么默认情况下，SpringMVC 会自动帮我们配置对应的 HttpMessageConverter，但是如果你使用的是 fastjson，则需要你自己手动配置。

    不过如果我的日期类使用频率很高，我希望它可以自动进行转换，可以在spring-servlet.xml文件中设置转换器。

静态资源访问

问题：由于我们在配置 DispatcherServlet 的时候，路径拦截规则是 / 表示拦截所有的请求，当然也包括静态资源请求。但是，静态资源请求不归 DispatcherServlet 处理。

  1. 不要使用 / 拦截所有请求（不推荐）。

    1. 首先拦截规则修改一下：

      <servlet-mapping>
    <servlet-name>springmvc</servlet-name>
    <url-pattern>*.action</url-pattern>
</servlet-mapping>

    2. 所有的接口定义，都必须以 .action 结尾：

      @GetMapping("/book.action")
@ResponseBody
public Book getBookByName(String name) {
    Book book = new Book();
    book.setName(name);
    book.setAuthor("123");
    book.setPublishDate(new Date());
    return book;
}

  2. 提供一个默认的静态资源处理器。

    <!--
默认的处理器
现在，所有的静态资源请求也会被拦截下来交给 DispatcherServlet 去处理，然而 DispatcherServlet 处理不了静态资源请求，当 DispatcherServlet 中的 Handler 处理不了这个请求的时候，那么就交给 DefaultHandler 去处理
-->
    <mvc:default-servlet-handler/>

  3. 配置静态资源放行策略

    <!--
mapping 就是请求路径
location 就是静态资源的位置
-->
<mvc:resources mapping="/**" location="/static/"/>

    其中使用到的路径匹配来自于一种风格

    Ant 风格的路径匹配符

    ?：代表单个字符
    *：代表一个具体的文件
    **：代表 N 层路径+一个文件

    最终的访问方式，就是将**所代表的字符，追加到 location 的后面

    例如我访问localhost:8080/html/01.html，文件实际路径为/WEB-INF/static/html/01.html，就可以匹配到。但是如果你访问localhost:8080/static/html/01.html，实际拼接出的路径就变成了/WEB-INF/static/static/html/01.html

全局异常处理

  1. 实现 HandlerExceptionResolver 接口：

    /**
 * 自定义类实现 HandlerExceptionResolver 接口，这个专门用来处理处理器中的异常
 */
@Component
public class MyGlobalException implements HandlerExceptionResolver {
    /**
     * 当处理器在执行的时候，抛出异常，那么就会进入到这个方法
     * @param request
     * @param response
     * @param handler 这个就是当前的处理器本身
     * @param ex 异常
     * @return
     */
    @Override
    public ModelAndView resolveException(HttpServletRequest request, HttpServletResponse response, Object handler, Exception ex) {
        ModelAndView mv = new ModelAndView("error");
        mv.addObject("msg", ex.getMessage());
        return mv;
    }
}

  2. 通过 @ControllerAdvice 来标记一个全局异常处理类，这种方式更灵活，可以返回视图也可以返回 JSON，可以服务端跳转，也可以客户端跳转。。。：

    /**
 * @ControllerAdvice 表示当前类是一个增强版的 Controller
 * 所以，这个类中方法的定义，就跟 Controller 中方法的定义一模一样
 */
@ControllerAdvice
public class MyGlobalException2 {

    /**
     * 这个异常处理方法的返回值，和接口中方法定义的规则一模一样
     *
     * @return
     * @ExceptionHandler(ArithmeticException.class) 注解表示当发生 ArithmeticException 异常的时候，就进入到当前方法中
     */
    @ResponseBody
    @ExceptionHandler(ArithmeticException.class)
    public Map<String, Object> arithmeticException(ArithmeticException e) {
        Map<String, Object> map = new HashMap<>();
        map.put("status", 500);
        map.put("msg", e.getMessage());
        return map;
    }
}

  3. 配置异常和视图之间的对应关系即可：

    <bean class="org.springframework.web.servlet.handler.SimpleMappingExceptionResolver">
    <property name="exceptionMappings">
        <props>
            <!--配置异常和视图的一一对应关系-->
            <prop key="java.lang.ArithmeticException">error</prop>
            <prop key="java.lang.NullPointerException">01</prop>
        </props>
    </property>
</bean>

文件上传

两种不同的方案：

  • 兼容老版本的 Tomcat（主要是指 Tomcat7 之前的版本），这种文件上传需要加一个依赖 common-fileupload。
  • 从 Tomcat7 开始，默认也可以处理文件上传，这种不用加依赖。

在 SpringMVC 中，也是两种文件上传方案：

  • CommonsMultipartResolver
  • StandardServletMultipartResolver

使用第二种方案做文件上传：

  1. 配置文件上传的临时目录等信息：

    <servlet>
    <servlet-name>springmvc</servlet-name>
    <servlet-class>org.springframework.web.servlet.DispatcherServlet</servlet-class>
    <init-param>
        <param-name>contextConfigLocation</param-name>
        <param-value>classpath:spring-servlet.xml</param-value>
    </init-param>
    <multipart-config>
        <!--文件上传的临时目录，这个目录不会自动创建，所以要给一个已经存在的目录-->
        <location>D:\workspace\temp</location>
        <!--上传文件的最大大小-->
        <max-file-size>1024000</max-file-size>
        <!--上传请求的最大大小，一个请求中可能包含多个上传文件-->
        <max-request-size>10240000</max-request-size>
        <!--上传文件的临界值-->
        <file-size-threshold>1024</file-size-threshold>
    </multipart-config>
</servlet>
<servlet-mapping>
    <servlet-name>springmvc</servlet-name>
    <url-pattern>/</url-pattern>
</servlet-mapping>

  2. 提供一个文件上传的解析器：

    <!--
这里只需要配置一个 Bean 即可，不需要额外提供任何属性
但是，这里的 id 必须是 multipartResolver，不能是其他名字，也不能没有名字
-->
<bean class="org.springframework.web.multipart.support.StandardServletMultipartResolver" id="multipartResolver">
</bean>

  3. 开发文件上传接口：

    @Controller
public class HelloController {

    SimpleDateFormat sdf = new SimpleDateFormat("/yyyy/MM/dd/");

    /**
     * OSS 对象存储服务
     * @return
     */
    @PostMapping("/upload")
    @ResponseBody
    public String uploadFile(MultipartFile file, String username, HttpServletRequest req) {
        System.out.println("username = " + username);
        //1. 先确定文件保存路径
        //获取项目运行路径
        String realPath = req.getServletContext().getRealPath("/img");
        String format = sdf.format(new Date());
        //文件保存的文件夹
        String s = realPath + format;
        System.out.println("s = " + s);
        File folder = new File(s);
        if (!folder.exists()) {
            //文件夹不存在就创建出来
            folder.mkdirs();
        }
        //2. 确定保存的文件名
        //获取原始的文件名
        String originalFilename = file.getOriginalFilename();
        //获取上传文件的后缀
        String suffix = originalFilename.substring(originalFilename.lastIndexOf("."));
        String newName = UUID.randomUUID().toString() + suffix;
        // 3. 保存文件
        try {
            file.transferTo(new File(folder, newName));
            //4. 返回文件上传路径
            return req.getScheme()//获取请求协议  http
                    + "://"
                    + req.getServerName()//获取项目域名 localhost
                    + ":"
                    + req.getServerPort()//获取项目端口 8080
                    + req.getContextPath()//获取项目名称 mvc05
                    + "/img"
                    + format
                    + newName;
        } catch (IOException e) {
            e.printStackTrace();
        }
        return "error";
    }
}

Servlet 中的下载方案：

// try-with-resources
@GetMapping("/download/{filename}")
public void download(@PathVariable String filename, HttpServletResponse resp) throws IOException {
    //加上这样一个响应头，就从文件访问变为文件下载了
    resp.setHeader("content-disposition","attachment;filename="
                + new String(filename.getBytes(StandardCharsets.UTF_8), StandardCharsets.ISO_8859_1));
    //获取文件的输入流
    try (FileInputStream fis = new FileInputStream("C:\\Users\\baize\\Pictures\\weixin\\" + filename)) {
        byte[] buf = new byte[1024];
        int len = 0;
        while ((len = fis.read(buf)) != -1) {
            resp.getOutputStream().write(buf, 0, len);
        }
    }
}

SpringMVC 中的下载方案:

@GetMapping("/download3/{filename}")
public ResponseEntity<byte[]> download3(@PathVariable String filename) throws Exception {
    //构建一个输出流，这个输出流将数据写出到 byte 数组中
    ByteArrayOutputStream baos = new ByteArrayOutputStream();
    try (FileInputStream fis = new FileInputStream("C:\\Users\\baize\\Pictures\\weixin\\" + filename)) {
        byte[] buf = new byte[1024];
        int len = 0;
        while ((len = fis.read(buf)) != -1) {
            baos.write(buf, 0, len);
        }
    }
    //1. 响应给前端的 byte 数组
    byte[] body = baos.toByteArray();
    //2. 响应头
    HttpHeaders header = new HttpHeaders();
    header.setContentDispositionFormData("filename", new String(filename.getBytes("UTF-8"),"ISO-8859-1"));
    //3. http 响应状态码
    return new ResponseEntity<byte[]>(body, header, HttpStatus.CREATED);
}

拦截器

类似于过滤器。

过滤器先执行，拦截器后执行。

/**
 * 自定义拦截器，类似于一种 AOP 风格的过滤器
 */
public class MyInterceptor implements HandlerInterceptor {

    /**
     * 这个是在处理器执行之前执行
     * @param handler 这个就是处理器，也就是接下来要执行的接口方法
     * @return 如果这个方法返回 false，那么请求就被拦截下来了，处理器将不再执行；请求如果返回 true，则当前请求继续向下走，处理器最终被执行
     */
    @Override
    public boolean preHandle(HttpServletRequest request, HttpServletResponse response, Object handler) throws Exception {
        System.out.println("preHandle");
        return true;
    }

    /**
     * 处理器执行之后会触发该方法
     * @param modelAndView 这个就是处理器执行完成之后，返回的 ModelAndView
     */
    @Override
    public void postHandle(HttpServletRequest request, HttpServletResponse response, Object handler, ModelAndView modelAndView) throws Exception {
        System.out.println("postHandle");
    }

    /**
     * 页面渲染完成之后，会触发的方法
     */
    @Override
    public void afterCompletion(HttpServletRequest request, HttpServletResponse response, Object handler, Exception ex) throws Exception {
        System.out.println("afterCompletion");
    }
}

    项目中，实际上包含两个不同的容器，Spring 容器和 SpringMVC 容器，Spring 容器是父容器，SpringMVC 容器是子容器，子容器中可以访问父容器中的 Bean，但是父容器无法访问到子容器中的 Bean。

    Controller 因为是在 DispatcherServlet 中使用的，所以 Controller 必须在 SpringMVC 容器中扫描

    一般来说，SpringMVC 中主要配置和视图相关的，例如控制器，视图解析器、参数解析等等。

    Spring 容器则主要配置业务相关的 Service、Mapper、事务、数据源。。。

    原因是spring配置文件里面context:exclude-filter是use-default-filters="true"，而context:include-filter是false，默认值就是false，不写即可。

实现WebApplicationInitializer接口，相当于 web.xml

/**
 * 这个类的作用，相当于 web.xml
 */
public class WebInit implements WebApplicationInitializer {
    /**
     * 当系统启动的时候，会自动执行这个方法，我们可以在这个方法中，完成容器的初始化
     * @param servletContext 使用到了servlet的包，导入依赖
     */
    public void onStartup(javax.servlet.ServletContext servletContext) throws ServletException {
        //创建一个 Spring 容器 //注意是web的，别写错，有个包很相似
        AnnotationConfigWebApplicationContext context = new AnnotationConfigWebApplicationContext();
        //将 servlet 容器设置给 Spring 容器
        context.setServletContext(servletContext);
        //让 Spring 容器去加载两个 Spring 相关的配置
        context.register(SpringConfig.class,SpringMVCConfig.class);
        ServletRegistration.Dynamic springMVC = servletContext.addServlet("springMVC", new DispatcherServlet(context));
        //设置 DispatcherServlet 的拦截规则
        springMVC.addMapping("/");
    }

}

serlvet如何匹配收到的服务请求：

  1. 路径精确匹配
  2. 最长路径匹配
  3. 扩展匹配

如果容器定义了一个default servelt（即匹配路径为/的servlet），则会将请求交给default servlet

注意

/*.action:错误的匹配规则，不能即是路径匹配，也是扩展匹配

spring mvc中dispatcherServlet配置为/*访问请求转发*.jsp和访问jsp页面的时候访问不到的原因：

默认在%TOMCAT_HOME%/conf/web.xml中配置了*.jsp由JspServelt来处理，当我们将spring mvc的拦截规则配置为/*的时候，==按照servlet的匹配规则，则路径匹配会优先于扩展匹配,导致对jsp的请求会被拦截掉。==

当spring mvc配置为/，会匹配到/login这样的路径型url，不会匹配到模式为*.jsp这样的后缀型url

当我们请求jsp时，刚好有从%TOMCAT_HOME%/conf/web.xml中继承过来的JspServlet会处理对jsp请求的处理，所以会访问到jsp页面。

springMVC项目访问404遇到的坑

项目一切都部署搞好了，可是就是无法访问，如果报上图信息，资源无法访问，显示jsp路径，那极有可能就是路径配置不对。

  1. 项目controller层加了类映射注解，但是spring-mvc.xml文件配置视图解析器的时候，路径写的是这样：

    @Controller
@RequestMapping("/book")
public class BookController {
    <bean class="org.springframework.web.servlet.view.InternalResourceViewResolver">
    <property name="prefix" value="WEB-INF/jsp/"/>
    <property name="suffix" value=".jsp"/>
</bean>

这是无法访问的，可去掉类映射注解，或者修改配置文件前缀为value="/WEB-INF/view/"，前面加/

RequestMapping默认的处理方法

不写就行了，不要画蛇添足。

@RestController
@RequestMapping("/home")
public class IndexController {
    @RequestMapping()
    String
    default () {
        return "This is a default method for the class";
    }
}
